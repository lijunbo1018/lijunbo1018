别忘写DOCTYPE

2017-01-08
学习笔记前端

遇到一个BUG，两页面采用完全相同的DOM结构和样式表，但是在浏览器里的表现就是不同。几番周折，发现唯一的不同就是其中一个页面首行没有写DOCTYPE。果不其然，加上DOCTYPE后，问题迎刃而解。那么这个熟悉又陌生的DOCTTYPE作用到底是什么？

DOCTYPE是什么

根据维基百科定义，DOCTYPE应该被称作文档类型声明（Document Type Declaration），是一条将特定的SGML或者XML文档与文档类型定义（Document Type Definition, DTD）相关联的指令。

From W3C：起初，DOCTYPE声明被用在基于SGML语言或者XML语言的HTML中。该声明通常是文档遵循标准实践的标识，在HTML5文档中，该声明用来区分“标准模式”和所谓的“怪癖模式”。无论是希望通过SGML或者XML解析器来解析HTML文档，还是要使文档在不同的HTML5处理器中被同样处理，DOCTYPE都是一个高质量HTML文档中至关重要的部分。由于历史原因，浏览器在解析以DOCTYPE声明开头的HTML文档时，会表现得更加可预测和更加统一。

值得注意的是，如果想要一个DOCTYPE能准确地匹配特定版本的(X)HTML，DOCTYPE声明必须出现在标签前，并且拼写、大小写都正确才能符合预期的工作。往往这是一个很长的指令，所以W3C列举了系列推荐的DOCTYPE声明供查阅。可以看到，HTML5文档的DOCTYPE声明最简洁：

至此，已经可以对DOCTYPE声明有基本的认识了。但是为什么HTML5的简洁而HTML4.01的冗长呢？延伸来学习的话，需要对上述几个名词做一个解释：

  • SGML: Standard Generalized Markup Language，标准通用标记语言，是用来定义通用标记语言的一个标准
  • XML: Extensible Markup Language，可扩展标记语言，它是SGML的一个子集，意在简化SGML
  • DTD: Document Type Difinition，文档类型定义，是SGML应用的一个特征，定义了标记结构的语法，还可能包含诸如字符实体引用的附加定义
  • 标准模式: standards mode，浏览器会根据标准来渲染页面和执行JavaScript
  • 怪癖模式: quirks mode，浏览器会采用更宽松的、向后兼容的（模仿旧浏览器的行为）方式来渲染页面和执行JavaScript，也称为混杂模式、兼容模式

HTML4是一个基于SGML定义的标记语言，即一个SGML应用；XHTML是一个基于XML定义的标记语言，即一个XML应用。它们都需要与一个特定的DTD关联，于是就有了上述指向dtd文件的URL。而HTML5不是基于SGML的，则不需要引用dtd文件。在HTML5中，它的存在用来开启“标准模式”。如果文档没有给出DOCTYPE声明或者DOCTYPE声明不合法，浏览器则会以“怪癖模式”来进行渲染。
